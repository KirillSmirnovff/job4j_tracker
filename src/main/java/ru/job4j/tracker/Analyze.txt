При просмотре через jmap и jstat видно, что при добавлении заявки создается один объект Item,
начинает утилизироваться больше Eden памяти, а при удалении зявки память не освобождается,
а объект Item соответственно не удаляется. При значительном заполнении Heap jvm пытается с помощью GC освободить хоть какое-то место,
удаляя в том числе объекты Item, которые были "удалены" в трекере и не имеют соответственно ссылок на себя.
При достижении ограничений по памяти jvm ложится с ошибкой:
"Exception in thread "main" java.lang.OutOfMemoryError: Java heap space".
